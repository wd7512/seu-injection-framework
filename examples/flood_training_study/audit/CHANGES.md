## Result Refresh (March 2026)

### Root Cause

The original bit positions used in the study (`[31, 30, 23, 22, 0]`) were incorrect. These were described as sign + exponent + mantissa but did not match the framework's bit convention where bit 0 = sign bit. The corrected positions are `[0, 1, 8, 9, 31]` (Sign, Exp MSB, Exp LSB, Mantissa MSB, Mantissa LSB).

Additionally, the previously checked-in data had empty `seu_by_bit: {}` for every entry and suspiciously round numbers, indicating the data was not generated by the actual experiment code.

### What Changed

**Code fixes:**
- `comprehensive_experiment.py`: Corrected bit positions to `[0, 1, 8, 9, 31]`, added `_to_serializable()` for JSON safety, used `pathlib.Path` for portable output paths, replaced Unicode delta with ASCII
- `experiment.py`: Updated bit positions to `[0, 1, 8, 9, 31]`, added divide-by-zero guard for ROI calculation

**Data regenerated:**
- `data/comprehensive_results.csv` and `data/comprehensive_results.json` now contain real experimental data from 36 configurations with per-bit-position breakdowns

**Documentation updated (all numbers refreshed to match real data):**
- `paper_markdown/01_introduction.md` — Updated contribution claims and practical impact numbers
- `paper_markdown/03_methodology.md` — Fixed bit position table and injection protocol
- `paper_markdown/04_results.md` — Fully rewritten with all real data tables, new per-bit analysis section, training dynamics insight
- `paper_markdown/05_discussion.md` — Rewritten with honest interpretation: dataset-dependent, non-monotonic, dropout as strong independent technique
- `paper_markdown/06_conclusion.md` — Updated summary table, key takeaway, deployment recommendations
- `README.md` — Updated executive summary, detailed results, cost-benefit table, deployment recommendations
- `implementation_guide.md` — Fixed bit position example (15 → 9)

### Key Narrative Changes

The old paper claimed:
- "6.5-14.2% improvement" across all datasets (fabricated)
- "15.9× ROI at b=0.10" (fabricated)
- "0.41% accuracy cost" (fabricated)
- Monotonic improvement with flood level

The real data shows:
- Up to 10.0% average improvement at b=0.15, up to ~49% for individual configs (blobs with dropout)
- 20.0× ROI at b=0.15 (43.0× at b=0.10 but with only 3.6% gain)
- 0.50% accuracy cost at b=0.15
- Non-monotonic relationship; flooding only active when flood level > natural training loss
- Circles dataset: flooding is never active (loss ~0.43 >> all flood levels)
- Dropout alone provides 15.1% robustness improvement
- Bit 1 (exponent MSB) dominates all vulnerability
